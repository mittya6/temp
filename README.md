# 背景
- 体制の肥大化による意思決定スピードの低下
- 「船頭多くして船山に上る」状態になってしまっている。

## 現状

### システム主管の作業範囲不透明
- 主管側のアウトプットが少なく、保守側に設計から作成まで負荷が集中。
- 結果として主管側が「チェッカー（評論家）」になってしまっている。

### 成果物が標準化されていない。
- レビューで後出しじゃんけん状態、観点バラバラ。
- レビューによる手戻り工数が多い。


## 解決案

### ドキュメントおよびテスト仕様の標準化
- 標準テンプレートの導入：プロパー決裁者の観点を盛り込んだテンプレートを作成。
- レビューでは「標準との乖離」「整合性」「表記ゆれ」のみを確認し、個人的な主観による追加要望は「別案件」として扱う。

### 役割分担の再定義（主管側へのタスク移管）
- システム主管の作業範囲やアウトプットを明確化させる。
- システム主管側の作業も手順書などドキュメント化しておく。

## 期待される効果
- リードタイムの短縮：レビュー回数の削減と、差し戻し率の低下。
- 作業の分担により生産性の効率化。
- リリースの安定化：事前に合意した「テスト合格基準」に基づき判断することで、根拠のないリリース延期を防止。
