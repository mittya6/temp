# 現状の「船頭多くして船山に上る」状態を脱却するために

## 背景
のっちがプロパーさん口説いて、大人数のシステム主管をアサインしたが機能していない。

## 現状の問題

### システム主管の作業範囲不透明
- システム主管が作成しているドキュメントやプログラムソースなどはほぼない。
- アウトプットのほとんどが保守が担っている。

### 成果物が標準化されていない。
- 価値観バラバラのシステム主管がレビューで後出しじゃんけん状態。
- 保守が作成するドキュメントも標準化されていないので、レビューによる手戻り工数が多い。




## 解決案

### 役割分担の再定義（主管側へのタスク移管）
- システム主管の作業範囲やアウトプットを明確化させる。
- 設計ドキュメントは主管、詳細手順は保守


### 
意思決定ラインの明確化させて、それに合わせたドキュメントを標準化させる。
