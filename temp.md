AWS EC2環境でのWindows Server 2012 R2のCドライブ拡張ですね。
この作業はEC2のダウンタイムなし（稼働中）で実行可能です。インフラ運用のセオリー通り、**作業前のスナップショット取得**を強く推奨します。

以下に、AWSマネジメントコンソール側とWindows OS側の2段階に分けた手順書を作成しました。

---

### 事前準備：バックアップの取得

1. AWSマネジメントコンソールで **EC2ダッシュボード** を開きます。
2. 左側のナビゲーションペインから **[スナップショット]** を選択し、**[スナップショットの作成]** をクリックします。
3. リソースタイプで「ボリューム」を選択し、対象インスタンスのルートボリュームIDを指定してスナップショットを作成します。

---

### ステップ1：AWS側でのEBSボリューム拡張

1. **EC2ダッシュボード** から **[インスタンス]** をクリックし、対象のWindows Server 2012 R2インスタンスを選択します。
2. 下部の詳細パネルから **[ストレージ]** タブを開き、ルートデバイス（通常は `C:` や `/dev/sda1` にマッピングされているもの）の **ボリュームID** をクリックします。
3. ボリュームの管理画面に遷移したら、対象ボリュームのチェックボックスを選択します。
4. 右上の **[アクション]** メニューから **[ボリュームの変更]** をクリックします。
5. 「サイズ (GiB)」の欄に、**現在のサイズに20を加えた数値**（例：現在が50GBなら「70」）を入力します。
6. **[変更]** をクリックし、確認のポップアップで再度 **[変更]** を選択します。

> **Note:** ボリュームの状態が「in-use - optimizing」になりますが、この状態でもステップ2のOS側での拡張作業は可能です（最適化完了までディスクI/Oパフォーマンスが若干低下する場合があります）。

---

### ステップ2：Windows OS側でのCドライブ拡張

1. 対象のEC2インスタンスに **RDP（リモートデスクトップ）** で接続します。
2. Windowsのスタートボタンを右クリック（または `Windowsキー + X`）し、**[ディスクの管理]** をクリックします。
* ※または「ファイル名を指定して実行」から `diskmgmt.msc` を実行します。


3. ディスクの管理画面が開き、ディスク0（Cドライブがあるディスク）の右側に **「未割り当て (Unallocated)」** の領域（約20GB）が追加されていることを確認します。
* *※未割り当て領域が表示されていない場合は、上部メニューの [操作] > [ディスクの再スキャン] をクリックしてください。*


4. **「C:」** のパーティションを右クリックし、**[ボリュームの拡張]** を選択します。
5. 「ボリュームの拡張ウィザード」が起動したら **[次へ]** をクリックします。
6. 「ディスクの選択」画面では、追加された未割り当て領域（20GB分のMB数）が自動的に選択されているため、そのまま **[次へ]** をクリックします。
7. **[完了]** をクリックしてウィザードを閉じます。

ディスクの管理画面およびエクスプローラー上で、Cドライブの容量が20GB増えていることを確認できれば作業は完了です。

---

**運用上のワンポイントアドバイス**
Windows Server 2012 R2は2023年10月に延長サポートが終了（EOS）しているため、AWS上であっても新規のセキュリティ更新プログラムは提供されていません。もし拡張サポート（ESU）等の特別な契約がない場合は、将来的なセキュリティリスクを考慮し、Windows Server 2022など新しいOSへの移行計画も並行して検討されることをおすすめします。
